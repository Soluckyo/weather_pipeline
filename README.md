# Weather Pipeline ETL

**Weather Pipeline** — это проект для сбора, обработки и анализа погодных данных с использованием ETL-пайплайна на Python и PostgreSQL.  
Проект структурирован по слоям **RAW → STG → DWH → MART**, поддерживает инкрементальную загрузку, data quality проверки и автоматизацию через Apache Airflow.

## Источник данных

- Данные подтягиваются из открытого и бесплатного API [Open-Meteo API](https://open-meteo.com/)

## Архитектура

### Слои

1. **RAW**  
   - Таблица raw_weather 
   - Данные загружаются через Python из API  
   - Логируются ошибки, повторные попытки при таймаутах  
   - Валидируются значения температуры, ветра, осадков

2. **STG (Staging)**  
   - Таблица stg_weather  
   - Преобразование времени (observation_time → obs_date + obs_hour)  
   - Убираются null значения  
   - Проверка дубликатов

3. **DWH (Data Warehouse)**  
   - **DIM**: dim_city 
   - **FACT**: fct_weather  
   - Инкрементальная загрузка с проверкой конфликтов

4. **MART (Витрины)**  
   - mart_city_stats — статистика по городам (средняя/минимальная/максимальная температура, средняя скорость ветра)  
   - mart_precipitation_daily — суммарные осадки по городам и датам
   - mart_avg_temp_daily - средняя температура по дням
